# OBJ → GLB 转换器 · 使用说明

把 **OBJ + MTL + 贴图（jpg/png）** 转换成一个**自包含的 GLB 文件**（贴图内嵌，单文件即可用于三维展示），支持**批量**。

---

## 一、最省事：用打包好的 exe（推荐给非技术使用者）

1. 拿到 `OBJ转GLB转换器.exe`（由技术同事用下面「打包」一节生成一次即可）。
2. **双击运行**，无需安装 Python。
3. 界面三步：
   - **第一步**：`添加文件…`（选一个或多个 `.obj`）或 `添加文件夹…`（把一整个目录里的模型一次性排队）。
   - **第二步**：选 `输出目录`（不选就默认放在输入目录下的 `output_glb`）。
   - **第三步**：设置贴图选项（默认即可），点 **开始转换**。
4. 完成后点 **打开输出目录** 查看结果。

> 每个 `.obj` 请和它的 `.mtl`、贴图放在同一文件夹；程序会自动带上。

---

## 二、没有 exe？直接用源码（需要 Python）

1. 安装 **Python 3.9+**（官网下载，安装时务必勾选 **Add Python to PATH**）。
2. 双击 **`install.bat`**（只需一次，自动安装 trimesh / pillow / numpy）。
3. 双击 **`run.bat`** 打开转换器。

---

## 三、贴图与几何选项说明

| 选项 | 作用 | 默认 |
|---|---|---|
| 贴图最大边长 | 超过则等比缩小（2048 对网页展示足够清晰、体积小） | 2048 |
| 贴图格式 | 自动＝有透明用 PNG、否则 JPEG | 自动 |
| JPEG 质量 | 数字越大越清晰、体积越大 | 85 |
| 不重编码贴图 | 最高画质：不动贴图编码，只按上限缩放 | 关 |
| 导出 transform.json | 生成每个模型的**归一化参数**（缩放/居中/落地），可直接填进展品配置 | 开 |
| 覆盖已存在 GLB | 同名文件是否覆盖 | 关 |

**关于压缩**：
- **几何：无损**（不做 Draco 压缩）。本类模型几何只占约 1/6 体积，压缩收益有限，故默认不压。
- **贴图：可选**。默认按上限缩放并重编码为 JPEG（有损、体积小）；勾「不重编码」即接近无损。
- 体积主要由贴图决定：一个 8192 贴图的模型，默认（2048）可从 ~6.9MB 降到 ~1.4MB。

**输出**：
- GLB 文件名带内容指纹（如 `模型-a1b2c3d4.glb`），内容一变文件名就变，方便 CDN 缓存。
- `manifest.csv`：所有模型的清单（体积、网格数、归一化参数、状态）。
- 每个模型的 `*.transform.json`：归一化参数。

**不支持 FBX**：请先在 Blender 里导出为 GLB 或 OBJ。

---

## 四、打包成 exe（技术同事操作，一次即可）

在一台装了 **Python 3.9+** 的 **Windows** 电脑上：

1. 把本文件夹（含 `obj2glb_gui.py`、`obj2glb.py`、`batch_glb.py`、`glb_utils.py`）整体拷过去。
2. 双击 **`build-exe.bat`**（自动装 PyInstaller 并打包）。
3. 产物在 `dist\OBJ转GLB转换器.exe`，把这个单文件发给使用者即可。

> exe 必须在 **Windows** 上打包（不能在 Linux/Mac 上生成 Windows exe）。

---

## 五、命令行版（可选，给会用命令行的人）

```bash
# 单个
python obj2glb.py 模型.obj --max-texture 2048
# 批量整目录
python batch_glb.py 输入目录 -o 输出目录 --max-texture 2048
```
