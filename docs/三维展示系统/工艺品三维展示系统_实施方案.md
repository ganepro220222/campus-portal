# 工艺品三维展示系统 · 实施方案

**项目**：云端书院小程序（贵州交通职业大学中华文化书院）
**模块**：非遗工艺品三维展示 | **版本**：V1.0

---

## 〇、术语约定

| 术语 | 含义 |
|------|------|
| **GLB** | glTF 2.0 的二进制封装格式。单文件，几何、材质、贴图全部内嵌，无外部依赖。三维交付的标准格式。 |
| **PBR** | 基于物理的渲染。材质由「粗糙度」「金属度」等物理参数描述，而非手工调色。 |
| **IBL** | 基于图像的光照。用一张全景图作为场景的光照与反射来源，物体表面会反射出这张图的内容。**"身临其境"效果的技术本体。** |
| **等距柱状投影** | Equirectangular。宽高比严格 2:1 的全景图格式，IBL 唯一接受的输入格式。 |
| **热点（Hotspot）** | 附着在模型表面某一坐标点的可点击标记，点击弹出图文讲解。 |
| **归一化** | 将不同来源、不同尺度的模型统一缩放居中，使展示时大小一致、位置端正。 |
| **XR-FRAME** | 微信小程序官方三维框架，可作为组件内联在小程序页面中。 |
| **Three.js** | Web 端主流三维引擎，运行在 H5 页面中，能力远强于 XR-FRAME。 |

## 一、需求精确化重述

原始需求经技术梳理后重新表述为四项。**后续开发以本节为准。**

### 需求 A：三维资产自主托管

**现状**：工艺品三维模型依赖第三方平台。将 obj/mtl/jpg 上传至第三方在线服务，由其生成含播放器的离线包（约 43MB），部署后得到一个 URL。资产与展示能力均受制于该平台。

**目标**：全部模型迁移至自有 OSS，由自有服务提供展示页面，彻底解除对第三方在线服务的依赖。

**验收**：第三方平台下线后，所有三维展示功能不受影响。

### 需求 B：三维资产生产管线自主化

**现状**：每新增一件工艺品，需上传至第三方后端处理，等待返回离线包。流程不可控、不可批量、不可离线。

**目标**：建立本地处理管线，支持三类输入的批量处理，无需任何外部服务：

1. 既有第三方离线包（zip）→ 提取其中现成的 GLB
2. 现成 GLB → 校验后直接使用
3. OBJ + MTL + 贴图 → 本地转换为自包含 GLB

处理过程同时完成**贴图压缩**与**归一化参数计算**，输出可直接入库的清单。

**验收**：一条命令处理上百个模型，输出 GLB 与 manifest.csv，全程无网络请求。

### 需求 C：三维展示能力升级

**现状**：第三方离线播放器仅支持自动旋转，无法调整光照，无法交互，无法标注。

**目标**：自建展示页，实现四项能力：

| 能力 | 精确定义 |
|------|---------|
| **C1 环境光照切换** | 用户可在多个预设环境间切换（展馆实景 / 影棚布光 / 晴天 / 阴天）。切换后模型的受光方向、高光形态、明暗对比随之改变。 |
| **C2 环境反射（核心）** | 模型表面按 PBR 规则反射当前环境。**当环境贴图使用真实展馆全景图时，工艺品釉面上会呈现展馆的射灯、展柜、墙面等实际细节**——即用户所述"身临其境""赛车鉴赏模式"的效果。 |
| **C3 交互控制** | 拖动任意角度旋转、滚轮/双指缩放、自动旋转开关、视角重置，带阻尼惯性。 |
| **C4 热点标注** | 模型表面可预置若干标记点。**默认隐藏**，用户主动开启后才显示，保证展示画面纯净。点击标记弹出该部位的图文讲解；标记随模型旋转，转至背面时自动淡化。 |

**验收**：切换环境时，模型表面反射内容随之改变，肉眼可辨。

### 需求 D：展馆 → 工艺品的跳转链路

**现状**：小程序展馆入口经 `web-view` 打开第三方 VR 展馆。展馆内工艺品热点当前指向第三方生成的离线包 URL。

**目标**：将 VR 展馆内工艺品热点的跳转目标，改为指向自有域名下的三维展示页。

```
小程序 → web-view → 第三方 VR 展馆 → 点击工艺品热点
       → https://shuyuan.gzcpu.edu.cn/craft/{id}   ← 自有服务
       → 从自有 OSS 加载 GLB + 全景图
```

**验收**：从展馆点击工艺品，进入自有展示页，功能完整。

## 三、总体架构

### 3.1 双入口设计（重要）

> **微信硬约束**：小程序中的 `<web-view>` 组件**独占整个页面**，无法与按钮、文字、价格等元素并存于同一屏。

因此**必须**保留两套展示实现，各司其职，**共用同一份 GLB 资产**：

| 入口 | 场景 | 技术实现 | 能力 |
|------|------|---------|------|
| **入口 1：商品详情页** | 需同屏显示【模型 + 商品名 + 价格 + 联系购买按钮】 | **XR-FRAME 内联组件**（已存在，保留） | 轻量预览：自转、缩放 |
| **入口 2：沉浸式鉴赏页** | 全屏无干扰的深度鉴赏 | **H5 + Three.js**（新建） | 完整能力：C1~C4 |

**连接方式**：

- 商品详情页底部增加「沉浸式鉴赏」按钮 → 全屏打开入口 2
- VR 展馆的工艺品热点 → 直接打开入口 2

## 四、关键技术约束（开发前必读）

以下每一条都会导致线上功能直接失效，**开发前逐条确认**。

### 约束 1：OSS 必须开启 CORS

Three.js 通过 `fetch` 加载 GLB 与全景图，属跨域请求。OSS 若未配置 CORS，浏览器直接拦截，模型加载失败。

```
来源(Origin)  : https://shuyuan.gzcpu.edu.cn
允许 Methods  : GET, HEAD
允许 Headers  : *
暴露 Headers  : ETag, Content-Length
缓存时间      : 86400
```

### 约束 2：三维资产用公共读 + CDN，不要用签名 URL

签名 URL 带时效参数，会**导致 CDN 与浏览器缓存全部失效**——同一个 3MB 模型每次访问都要重新下载。

三维模型属公开展示内容，无隐私风险。**Bucket 设为公共读，绑定 CDN，设置长缓存**（`Cache-Control: max-age=31536000`），文件名带内容哈希以支持更新。

> **与现网差异（实施前需决策）**：当前 `OssService` 对多数媒体走**私有 Bucket + 签名 URL**；`CraftService` 的 `model3dUrl` 目前原样下发、未签名。H5 鉴赏页若用 Three.js `fetch` 跨域加载，需满足其一：① 模型走公共读 CDN 前缀（本方案推荐）；② 或后端下发**长有效期**签名 URL 并接受缓存打折。不要与封面图/视频混用同一套短时效签名策略。

### 约束 3：热点坐标必须存「模型局部坐标」

若存世界坐标，模型一旋转，热点立即错位飞出。

```javascript
// ✗ 错误：世界坐标
const pos = intersects[0].point;

// ✓ 正确：转为模型局部坐标
const pos = model.worldToLocal(intersects[0].point.clone());
```

渲染时再转回世界坐标：

```javascript
worldPos.copy(hotspot.localPos).applyMatrix4(model.matrixWorld);
```

### 约束 4：GLB 贴图必须内嵌

若 GLB 的贴图为外部引用（`images[].uri` 指向外部文件），上传 OSS 后路径失效，模型显示为纯白。

**入库前必须校验**：解析 GLB 的 JSON 块，确认所有 `images[]` 均含 `bufferView` 字段（内嵌），而非 `uri`。批处理脚本已实现此校验。

### 约束 5：全景图必须是等距柱状投影（2:1）

IBL 只接受宽高比严格 2:1 的等距柱状全景图。VR 平台内部常存储为**立方体六面图**或**瓦片金字塔**，不能直接使用。

**必须向 VR 服务商索要原始全景图。** 若无法获取，C2（反射真实展馆）无法实现，只能退化为通用预设环境。

### 约束 6：`shuyuan.gzcpu.edu.cn` 必须加入微信「业务域名」

小程序 `web-view` 只能打开业务域名白名单内的页面。此项已列入《上线分工 Checklist》§⑤。

### 约束 7：模型必须在入库时完成归一化

不同来源的 GLB 尺度与原点差异极大（有的 1 单位 = 1 米，有的 = 1 厘米；有的原点在底部，有的在几何中心）。直接加载会出现"模型飞出屏幕"或"小到看不见"。

归一化参数在**上传时计算一次**存库，前端加载时套用。**不要在前端每次实时计算**——会拖慢首屏。

## 七、分步实施计划

### 阶段 1：资产处理管线（1 天，可独立交付，无前置依赖）

**目标**：把现有全部工艺品模型处理出来，摸清家底。

1. 部署 `batch_glb.py` 批处理脚本
2. 收集所有工艺品源文件（离线包 zip / GLB / OBJ）放入同一目录
3. 执行批处理：
   ```bash
   python batch_glb.py 工艺品源文件 -o 处理结果 --max-texture 2048
   ```
4. 检查 `manifest.csv`，重点关注：
   - **「贴图内嵌」为「否」的条目** → 需单独修复，否则线上贴图丢失
   - **「状态」为失败的条目** → 排查源文件问题
   - 体积超过 8MB 的模型 → 考虑降低贴图尺寸

**交付物**：`处理结果/*.glb` + `manifest.csv`（含 scale / offsetX / offsetY / offsetZ，可直接入库）

**此阶段不依赖任何其他工作，可立即开始。**

### 阶段 3：H5 沉浸式鉴赏页（2 天）

**技术栈**：Three.js（原生，不引入框架），单页部署至 `/craft/{id}`

```javascript
// ── 1. 加载 GLB 并套用归一化参数 ──
const loader = new THREE.GLTFLoader();
loader.load(config.modelUrl, (gltf) => {
  const model = gltf.scene;
  const t = config.transform;
  model.scale.setScalar(t.scale);
  model.position.set(t.offsetX, t.offsetY, t.offsetZ);

  // 套用材质参数（覆盖 GLB 内置设置）
  model.traverse((child) => {
    if (child.isMesh && child.material) {
      child.material.roughness       = config.material.roughness;
      child.material.metalness       = config.material.metalness;
      child.material.envMapIntensity = config.material.envMapIntensity;
      child.material.needsUpdate     = true;
    }
  });
  scene.add(model);
});

// ── 2. 环境贴图（IBL）—— 反射效果的本体 ──
const pmrem = new THREE.PMREMGenerator(renderer);
pmrem.compileEquirectangularShader();

function applyEnvironment(preset) {
  new THREE.TextureLoader().load(preset.panoramaUrl, (tex) => {
    tex.mapping = THREE.EquirectangularReflectionMapping;
    scene.environment = pmrem.fromEquirectangular(tex).texture;  // 模型反射它
    scene.background  = new THREE.Color(preset.bgColor);
    renderer.toneMappingExposure = preset.exposure;
    tex.dispose();
  });
}

// ── 3. 色调映射（必须开，否则高光过曝、廉价感）──
renderer.toneMapping    = THREE.ACESFilmicToneMapping;
renderer.outputEncoding = THREE.sRGBEncoding;

// ── 4. 热点：局部坐标 → 世界坐标 → 屏幕坐标 ──
function updateHotspot(spot) {
  worldPos.copy(spot.localPos).applyMatrix4(model.matrixWorld);
  const p = worldPos.clone().project(camera);
  spot.el.style.left = (p.x * 0.5 + 0.5) * innerWidth  + 'px';
  spot.el.style.top  = (-p.y * 0.5 + 0.5) * innerHeight + 'px';

  // 遮挡判定：射线击中模型且距离更近 → 热点在背面 → 淡化
  raycaster.set(camera.position,
                worldPos.clone().sub(camera.position).normalize());
  const hits = raycaster.intersectObject(model, true);
  const occluded = hits.length > 0
    && hits[0].distance < camera.position.distanceTo(worldPos) - 0.06;
  spot.el.classList.toggle('behind', occluded);
}
```

**UI 规范**（沿用项目视觉基线）：

- 主色 `#2B356E`（藏青），点缀色 `#BE9C44`（金）
- 「细节标注」**默认关闭**，保证展示画面纯净
- 加载期显示 `posterUrl` 封面 + 进度条，避免白屏

**参考实现**：配套的 `工艺品立体鉴赏_演示.html` 已实现全部交互逻辑，将程序生成的模型与环境替换为真实 GLB 与全景图即可。

### 阶段 4：管理端热点编辑器（2.5 天，最难）

**这是整个项目唯一的硬骨头。**

**功能**：管理员在后台看到可旋转的三维模型，按住 `Shift` 点击模型表面即可打点，填写标题与正文，保存。

```javascript
// ── 打点：屏幕点击 → 射线检测 → 模型局部坐标 ──
canvas.addEventListener('click', (e) => {
  if (!e.shiftKey) return;   // Shift 才是打点，否则为旋转视角

  const rect = canvas.getBoundingClientRect();
  mouse.x =  ((e.clientX - rect.left) / rect.width)  * 2 - 1;
  mouse.y = -((e.clientY - rect.top)  / rect.height) * 2 + 1;

  raycaster.setFromCamera(mouse, camera);
  const hits = raycaster.intersectObject(model, true);
  if (!hits.length) return;

  // 关键：必须转为模型局部坐标（见约束 3）
  const local = model.worldToLocal(hits[0].point.clone());

  openHotspotDialog({
    x: +local.x.toFixed(5),
    y: +local.y.toFixed(5),
    z: +local.z.toFixed(5)
  });
});
```

**编辑器需具备**：

- 左侧：三维视图（可旋转、缩放、Shift+点击打点）
- 右侧：热点列表（点击定位、编辑、删除、拖拽排序）
- 底部：材质参数滑块（粗糙度 / 金属度 / 反射强度）+ 环境预设下拉 + 实时预览
- 「保存机位」按钮：把当前视角存为 `camera_json`

**注意**：管理后台是 Vue 3 + Element Plus。Three.js 部分封装为独立组件 `ModelEditor.vue`，与业务逻辑解耦（遵循高内聚低耦合原则）。

## 十、注意事项与风险

### 10.1 性能

| 项 | 建议 |
|----|------|
| 单个 GLB 体积 | 控制在 **3MB 以内**。贴图压至 2048 通常可将 4096 贴图的模型从 2.2MB 降至 1.2MB |
| 全景图体积 | 控制在 **1.5MB 以内**（2048×1024 JPEG，质量 85） |
| 首屏体验 | 加载期显示封面图 + 进度条，禁止白屏 |
| 缓存 | CDN 长缓存（1 年），文件名带内容哈希，更新时换名 |
| 移动端 | `renderer.setPixelRatio(Math.min(devicePixelRatio, 2))`，防止高分屏渲染压力过大 |

### 10.2 质感调校经验值

不同材质的 PBR 参数差异极大，用错会很难看：

| 材质 | roughness | metalness | envMapIntensity |
|------|-----------|-----------|-----------------|
| 陶瓷（釉面） | 0.10 ~ 0.20 | 0.00 ~ 0.05 | 1.2 ~ 1.8 |
| 陶器（无釉） | 0.75 ~ 0.90 | 0.00 | 0.4 ~ 0.7 |
| 青铜 / 银器 | 0.25 ~ 0.45 | 0.80 ~ 1.00 | 1.5 ~ 2.5 |
| 木器 | 0.60 ~ 0.85 | 0.00 | 0.5 ~ 0.8 |
| 玉石 | 0.15 ~ 0.30 | 0.00 | 1.0 ~ 1.5 |

### 10.3 热点标注的工作量控制

上百件工艺品**不必全部配置热点**：

- 挑选 **10~20 件**有讲解价值的重点展品配置热点
- 其余仅做纯三维展示（自转 + 缩放 + 环境反射），观感依然优良
- 热点文案可交由校方老师在后台自行录入——他们比开发者更懂文物讲解

### 10.4 开发规范

严格遵守项目三条原则：

1. **高内聚低耦合**：Three.js 渲染逻辑独立成模块；管理端 `ModelEditor.vue` 单独封装
2. **代码注释全部使用中文**
3. **不出现任何第三方工具、模型、平台的名称或痕迹**

### 10.5 降级策略

| 场景 | 降级方案 |
|------|---------|
| 设备不支持 WebGL | 显示 `posterUrl` 静态封面 + 提示 |
| GLB 加载失败 | 显示封面图 + 「重新加载」按钮 |
| 全景图加载失败 | 回退至内置的程序生成环境（不影响模型显示） |
| 无热点数据 | 不显示「细节标注」开关 |
| 无环境预设 | 不显示环境切换栏，使用默认影棚环境 |
