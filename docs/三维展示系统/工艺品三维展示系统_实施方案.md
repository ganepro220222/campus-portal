# VR 展馆工艺品静态 3D 展包方案（开发方内部）

> **产品边界（已定稿）**
>
> | 角色 | 职责 |
> |------|------|
> | **校方后台** | 维护文创名称、中英文介绍、多角度鉴赏图、咨询联系方式、上下架 |
> | **开发方** | 批处理 GLB、打包静态 H5、部署至服务器、在 VR 平台配置热点外链 |
>
> 小程序「文化好物」**不包含** 3D 模型展示；`craft` 表**不存** GLB URL、viewer 配置。  
> 后端**不提供** `/crafts/{id}/viewer`、后台 GLB 上传、`viewer-config` 等动态接口。

---

## 1. 术语

| 术语 | 说明 |
|------|------|
| **GLB** | glTF 2.0 二进制封装，几何+材质+贴图内嵌，静态展包标准格式 |
| **静态展包** | `index.html` + `model.glb` +（可选）`panorama.jpg` + `config.json`，整目录部署 |
| **VR 热点** | VR 展馆平台中的可点击区域，外链指向静态展包 URL |

---

## 2. 资产流水线（Python，保留）

```
源资产（OBJ/zip/GLB）
    → docs/三维展示系统/batch_glb.py
    → test/output_glb/*.glb + manifest.csv
    → 开发方组装展包目录
    → SSH/OSS 上传至 /exhibits/craft-{id}/
    → VR 平台配置热点 URL
```

| 脚本 | 用途 |
|------|------|
| `glb_utils.py` | 校验、归一化、贴图内嵌检查 |
| `batch_glb.py` | 批量 OBJ/zip/GLB → 输出 GLB + manifest |
| `obj2glb.py` | 单文件转换 |

本地素材与输出：`test/1`、`test/7`、`test/8`、`test/output_glb/`。  
操作说明见 `test/README.md`。

---

## 3. 静态 H5 展包（保留与扩展）

| 路径 | 说明 |
|------|------|
| `test/4/index.html` | 当前原型：Three.js + RoomEnvironment PMREM + 调参面板 |
| `docs/三维展示系统/工艺品立体鉴赏_演示.html` | 设计参考：热点 + 全景环境切换 |

**Phase 2 待实现（VR 展馆必做）：**

1. **全景 IBL**：equirectangular `panorama.jpg` → PMREM，模型随环境光照变化  
2. **热点**：`config.json` 定义 3D 空间锚点 + 文案/跳转  
3. **展包 schema**：`config.json` 统一 transform、材质、相机初值

部署 URL 示例：`https://shuyuan.gzcpu.edu.cn/exhibits/craft-001/index.html`  
热点模板：`scripts/vr-craft-hotlink.example.txt`

## 4. 与小程序 / 后端的关系

| 能力 | 位置 |
|------|------|
| 文创列表/详情（多图） | 小程序 `packageA/craft` + `GET /api/v1/crafts` |
| 720° 展馆 VR | `hall.vr_url` → 外部全景（与 GLB 展包独立） |
| 工艺品 GLB 3D 鉴赏 | **仅** VR 热点 → 静态 H5 |

已删除（勿恢复）：

- 小程序 `xr-frame` / `viewer-webview`
- 动态 H5 `viewer/` 项目
- `CraftViewerService`、Java GLB 上传校验
- `craft` 表 `model_3d_url`、`viewer_enabled`、`transform_json` 等列

旧库清理：执行 `sql/patch-remove-craft-3d-columns.sql`。

---

## 5. 质量与体积

| 指标 | 建议 |
|------|------|
| 单 GLB | ≤ 3MB；贴图 2048 JPEG q≈85 |
| 贴图 | 必须内嵌 GLB，禁止外链 uri |
| 复杂 PBR | 优先直接交付 GLB（Blender 导出），OBJ 管线仅适合基础漫反射 |
