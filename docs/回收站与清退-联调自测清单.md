# 回收站 / 清退 联调自测清单

> 覆盖「内容删除→回收站→恢复/彻底删除」与「师生清退（匿名化）」两条链路。
> 前置：以超级管理员（`admin:super`）登录后台；如为旧库先执行 `sql/patch-recycle-bin.sql`（新库无需）。

## 一、内容删除 → 回收站

对 展馆 / 文创 / 课程 / 资源 / 新闻 / 活动 六类分别验证：

- [ ] **上架/已发布态看不到「删除」**：列表操作列在内容处于上架（活动为「已发布」）时不出现「删除」，只有下架/草稿/已取消/已结束态才出现，符合「先下架再删除」。
- [ ] **删除入回收站**：点「删除」→ 确认后提示「已移入回收站」，列表该项消失。
- [ ] **小程序端不可见**：删除后小程序对应模块 / 搜索中不再出现该内容（删除前已下架，故本就不可见——确认没有「幽灵残留」）。
- [ ] **回收站可见**：进入「内容运营 → 回收站」，切到对应类型标签，能看到刚删除项，标签角标计数 +1，显示名称与删除时间。

## 二、恢复

- [ ] 在回收站点「恢复」→ 确认后提示「已恢复」，该项从回收站消失、角标 -1。
- [ ] 回到对应内容列表，该项重新出现，且为**下架 / 草稿**状态（不是自动上架）。
- [ ] 手动上架后，小程序端 / 搜索恢复可见，数据（封面、图文、子内容如展馆分区/文创图片）完整无缺。

## 三、彻底删除（含引用拦截）

- [ ] **无引用可彻底删除**：对一条没有报名/收藏/点赞/下载/学习记录的回收站项点「彻底删除」→ 确认后提示「已彻底删除」，数据库中该行消失。
- [ ] **展馆/文创级联**：彻底删除展馆后，其 `hall_section`/`hall_media` 子行一并物理清除；文创的 `craft_image`/`craft_contact` 同理；不留孤儿行。
- [ ] **引用拦截 - 活动**：对有报名记录（`enroll`）的已删活动点「彻底删除」→ 被拦截，提示形如「仍存在 N 条报名记录，无法彻底删除」，数据未被删。
- [ ] **引用拦截 - 资源**：对有下载记录（`download_record`）的资源同样被拦截。
- [ ] **引用拦截 - 课程**：对有学习记录（`course_progress`）的课程被拦截。
- [ ] **引用拦截 - 收藏/点赞**：任一类型只要存在 `favorite`/`like_record` 引用即被拦截。
- [ ] 拦截后该项仍留在回收站，可继续「恢复」。

## 四、师生清退（匿名化）

- [ ] 「用户服务 →（内容运营下）师生账号」列表，正常账号操作列出现「清退」；已清退账号显示「已清退」标签、无「清退」按钮。
- [ ] 点「清退」→ 弹窗明确告知「脱敏姓名/学号/手机号并禁止登录、历史记录保留、不可恢复」，确认按钮为危险红色。
- [ ] 确认后：姓名显示「已清退」、学号清空、手机号清空、状态变「禁用」。
- [ ] **登录失效**：该生原学号密码无法登录；若此前持有 App token，接口应因 `token_version` 递增而 401（需重新登录）。
- [ ] **统计完整性**：该生历史 `enroll`/`point_record`/`download_record` 等仍在，报名人数、积分榜等统计不减少、不报错。
- [ ] **学号释放**：清退后同一学号可再次通过 Excel 导入创建为新账号（不与旧匿名账号冲突）。

## 五、权限与回归

- [ ] 非超级管理员看不到「回收站」菜单；直接访问 `/recycle-bin` 被重定向到首页概览。
- [ ] 各内容「删除」按钮受原有写权限约束（新闻 `news:write`、展馆/文创 `hall:write`、课程/资源 `course:write`、活动 `admin:super`）。
- [ ] 原有增删改查、上下架、发布、报名管理等功能不受影响（冒烟一遍）。
