# CAS / 统一身份认证对接评估说明

## 背景

当前小程序登录方案为：

1. **校方 Excel 批量导入**学号账号（初始密码 + 首次改密）
2. **学号密码登录** + **微信绑定**
3. 管理端独立账号体系（与师生分离）

随着学校信息化成熟，常希望对接 **CAS（Central Authentication Service）** 或厂商统一身份认证，实现「用学校门户账号直接登录」。

## 对接价值

| 收益 | 说明 |
|------|------|
| 免维护师生密码 | 密码策略、找回、过期由学校 IdP 负责 |
| 免 Excel 同步 | 登录时实时验权，减少名册滞后 |
| 体验一致 | 师生已熟悉学校统一登录入口 |
| 安全合规 | 符合高校等保与账号集中管理要求 |

## 对接成本与前提

| 项 | 说明 |
|----|------|
| 学校配合 | 提供 CAS 地址、协议版本、测试账号、回调白名单 |
| 网络 | 后端须能访问学校 CAS（常需 VPN 或政务/教育网） |
| 用户标识 | 需约定唯一字段（学号/工号）与 `member` 表映射规则 |
| 微信场景 | 小程序不能浏览器重定向 CAS，需 **后端代理票据校验** 或 **H5 中转页** |

## 技术路线对比

### 方案 A：后端 CAS Ticket 校验（推荐评估）

```
小程序 → 学校 CAS 登录页(H5 webview) → 带 ticket 回调后端
     → 后端 /auth/cas/validate → 签发本平台 JWT → 小程序登录
```

- **优点**：与现有 JWT 体系一致；微信绑定逻辑可保留
- **缺点**：需 webview 页与学校 CAS 回调域名备案
- **工作量**：约 3–5 人日（含联调），视学校 CAS 文档质量浮动

### 方案 B：OAuth2 / OIDC（若学校提供）

部分高校新版身份平台支持 OIDC，可用 `authorization_code` 换 token，再取 `sub`/学号。

- **优点**：标准协议，文档相对完善
- **缺点**：并非所有学校 CAS 都支持 OIDC
- **工作量**：约 3–6 人日

### 方案 C：仅管理端 CAS，小程序仍 Excel（折中）

管理员用学校账号登录后台；师生端暂不改。

- **优点**：改动小，先解决运维账号统一
- **缺点**：不解决学生免导入诉求

## 与现有导入方案的关系

| 模式 | 适用 |
|------|------|
| **仅 CAS** | 学校禁止本地密码、名册实时查 LDAP/人事库 |
| **CAS + 导入** | CAS 登录后仍须匹配预导入名册（控制准入范围） |
| **仅导入** | 试点、分院、无 CAS 条件时（**当前实现**） |

推荐生产：**CAS 负责认证，导入或教务 API 负责准入名单**（防止校外人员注册）。

## 微信绑定在 CAS 模式下

CAS 登录成功后，若 `openid` 仍为占位符，可沿用现有 `wx-bind` 流程；或在学校 IdP 关联微信（较少见）。

## 实施建议（分阶段）

### 阶段 1（当前已完成）

- Excel 导入 + 列名别名 + 失败明细导出
- 学号登录 + 微信绑定 + 首次改密

### 阶段 2（学校确认 CAS 信息后进行）

1. 学校提供测试环境 CAS URL、service 参数规范
2. 新增 `CasAuthService`、`/auth/cas/login-url`、`/auth/cas/callback`
3. 小程序 webview 页完成票据回传
4. 联调 + 压测 + 回退开关（`auth.mode=cas|local`）

### 阶段 3（可选）

- 教务系统 API 定时同步，替代 Excel
- 教职工 LDAP 区分学生/教师角色

## 风险与回退

- **CAS 故障**：配置降级为学号本地登录（保留导入账号）
- **学号不一致**：需映射表或导入时写入 `external_id`
- **毕业/退学**：依赖学校账号注销或定期同步禁用状态

## 结论

- **短期**：Excel 导入 + 表头别名已可满足大规模师生入库，**无需全校手抄模板**。
- **中期**：建议与学校信息中心确认是否提供 CAS/OIDC；若提供，优先 **方案 A** 与现有 JWT 融合。
- **决策输入**：请学校提供 CAS 版本、测试环境、是否要求师生必须走统一认证、是否仍需要准入名单。

如需立项 CAS 对接，请学校侧填写：CAS 基址、回调域名、学号字段名、是否支持 OIDC、测试账号 2 个。
