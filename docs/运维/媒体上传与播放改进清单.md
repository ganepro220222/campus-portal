## 二、上传与规格（待对齐 / 增强）

| 优先级 | 项 | 现状 | 建议 |
|--------|-----|------|------|
| P1 | 文档与代码大小上限一致 | 代码默认 **200MB** 中转；文档写图片 20MB、视频 2GB | 区分场景上限，或接入 OSS 直传后放开视频 |
| P1 | OSS 直传（STS） | 后台中转上传，大视频易超时占带宽 | 按技术方案 §4.4 接前端直传 |
| P1 | 视频异步转码 | 未接入 MPS/VOD | 上传后转 720p H.264，统一码率与体积 |
| P2 | 上传前体积/尺寸校验 | 仅后端拒绝超大文件 | 前端选择文件后即时提示 MB 与建议分辨率 |
| P2 | OSS 图片处理 | 文档有 w_400/w_1200 缩略 | 列表接口返回带处理参数的 CDN URL，减流量 |
| P3 | 展馆/文创画廊图展示模式 | 仅封面支持 fit/fill | 可按需扩展 `hall_media` 单图模式 |

## 四、学习进度统计（待严谨化）

| 优先级 | 项 | 现状 | 建议 |
|--------|-----|------|------|
| P1 | 完成判定 | 播放位置 ≥ 90% 即完成 | 验收已满足；教务级需加强 |
| P2 | 防拖拽刷进度 | 客户端上报，可拖到 90% | 累计有效观看时长或关键帧打卡 |
| P2 | 完成状态不回退 | 低进度上报可能把 `completed` 改回 0 | 完成后仅允许进度前进 |
| P2 | 时长交叉校验 | `duration` 来自播放器 | 与课程元数据 / 视频元信息比对 |
| P3 | 游客学习记录 | 未登录不上报 | 登录后合并或提示登录保存进度 |

当前机制（已实现）：

- 每 **20 秒**上报；退出/暂停补报
- **断点续播**（`initial-time`）
- 完成 **≥90%** 触发积分（不重复发放）

## 五、内容同事上传备忘

| 用途 | 建议比例 | 展示模式建议 |
|------|----------|--------------|
| 课程/文创列表封面 | 1:1 或 4:3 | 默认裁切填满；竖图选完整显示 |
| 新闻列表封面 | 4:3 或 16:9 | 主体居中 |
| 展馆列表封面 | 4:3 或 16:9 | 主体居中 |
| 活动顶图 | 16:9 或 2:1 | 横图 |
| 首页 Banner | 约 2:1 | 横图，文字留安全区 |
| 课程教学视频 | **16:9** | 播放器完整显示，非 16:9 会有黑边 |
| 新闻正文插图 | 任意 | 宽度自适应，最宽容 |
