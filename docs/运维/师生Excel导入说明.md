# 师生 Excel 导入说明

## 是否需要固定格式？

**需要遵循列含义，但不必逐格手抄到空白模板。**

系统提供「下载导入模板」作为字段说明与示例，但校方若已有内部师生名册，通常只需在 Excel 里**改表头列名**（或保留常见别名），即可直接上传导入。

## 必填与可选字段

| 字段 | 是否必填 | 说明 |
|------|----------|------|
| 学号 | **必填** | 全校唯一；已存在则跳过 |
| 姓名 | **必填** | 显示名称 |
| 学院 | 可选 | 用于个人资料展示 |
| 年级 | 可选 | 如 2024 |
| 手机号 | 可选 | |
| 身份证号 | 可选 | 用于生成初始密码（后 6 位）；无则取学号后 6 位 |

单次导入上限 **5000 行**；格式 `.xlsx` / `.xls`。

## 支持的表头别名（改列名即可）

校方表头与下列任一名称一致即可被识别（**列顺序不限**，多余列可保留）：

| 系统字段 | 可识别的表头 |
|----------|----------------|
| 学号 | 学号、学生学号、学号/工号、工号、账号、student_no |
| 姓名 | 姓名、学生姓名、名字、真实姓名、name |
| 学院 | 学院、院系、所在学院、学院名称、college |
| 年级 | 年级、入学年级、grade |
| 手机号 | 手机号、手机、联系电话、电话、mobile、phone |
| 身份证 | 身份证号、身份证、证件号码、身份证号码、id_card |

### 推荐操作（无需手抄）

1. 打开校方导出的 Excel（教务/学工系统导出均可）。
2. 将第一行表头改为上表对应名称（如「院系」→「学院」）。
3. 删除与导入无关的列（可选，不影响导入）。
4. 在管理后台「师生账号」页上传。

若表头差异较大，可用 Excel 公式引用原表数据到新表（如 `=Sheet1!B2`），**批量引用比手工抄写快得多**。

## 导入后账号规则

- **初始密码**：身份证后 6 位；无身份证则学号后 6 位。
- **首次登录**：学生须在小程序修改初始密码后方可正常使用。
- **微信登录**：须绑定校方导入的学号账号（不自动注册新用户）。

## 导入失败怎么办？

导入结果会显示成功/跳过/失败数量。若有失败行：

1. 弹窗内可查看前 50 条错误摘要；
2. 点击「**导出上次失败明细**」下载 Excel，含行号、学号、姓名、失败原因；
3. 修正后重新导入（已成功的学号会自动跳过）。

## 常见失败原因

- 学号或姓名为空
- 学号已存在（显示为「跳过」，非失败）
- 文件非 Excel 或超过 5000 行

## 后续可增强（按需）

若某校表头与上述别名仍不匹配，可评估：

- 管理端「列映射」配置（上传后手动对应列）
- 与教务系统 API 对接（免 Excel）
- CAS / 统一身份认证单点登录（见《CAS 对接评估说明》）
