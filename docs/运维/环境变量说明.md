# 环境变量说明

> Phase E1-3 | 基线提交 `3e1f9d5`  
> 本文档描述 dev / docker / staging / prod / test 各环境的配置边界与变量规范。

## 1. Profile 一览

| Profile | 配置文件 | 用途 | 敏感信息 | Mock / dev-mode |
|---------|----------|------|----------|-----------------|
| `dev` | `application-dev.yml` | 本地 IDE 开发 | 可含 dev 密码 | `wx.dev-mode=true` |
| `docker` | `application-docker.yml` | Docker Compose | compose 内 dev 密码 | `wx.dev-mode=true`（继承 base） |
| `staging` | `application-staging.yml` | 预发验证 | **必须**环境变量 | `wx.dev-mode=false` |
| `prod` | `application-prod.yml`（本地，不入库） | 正式生产 | **必须**环境变量 | `wx.dev-mode=false` |
| `test` | `src/test/resources/application-test.yml` | 单测/集成测 | 测试库密码 | `wx.dev-mode=true` |

### 激活方式

```bash
# 本地默认 dev（application.yaml 默认值）
# 无需设置环境变量即可启动

# 指定 profile
export SPRING_PROFILES_ACTIVE=staging   # Linux/macOS
set SPRING_PROFILES_ACTIVE=staging    # Windows CMD
$env:SPRING_PROFILES_ACTIVE="staging"  # Windows PowerShell

# Docker Compose 已在 docker-compose.dev.yml 中设置 SPRING_PROFILES_ACTIVE=docker
```

## 2. 环境变量清单

### 2.1 通用

| 变量名 | 必填 | 默认值 | 说明 |
|--------|------|--------|------|
| `SPRING_PROFILES_ACTIVE` | 否 | `dev` | Spring 激活的 profile。**staging/prod 部署必须改为 `staging` 或 `prod`**，否则绕过启动门禁 |
| `SERVER_PORT` | 否 | `8080` | HTTP 端口 |
| `APP_VERSION` | 否 | `0.0.1-SNAPSHOT` | 应用版本，health 接口与 CI 可读取 |

### 2.2 数据库（staging / prod 必填）

| 变量名 | 示例 | 说明 |
|--------|------|------|
| `DB_URL` | `jdbc:mysql://db.example.edu.cn:3306/shuyuan?...` | MySQL JDBC 连接串 |
| `DB_USERNAME` | `shuyuan_app` | 数据库用户名 |
| `DB_PASSWORD` | （随机强密码） | 数据库密码 |

### 2.3 Redis（staging / prod 必填）

| 变量名 | 示例 | 说明 |
|--------|------|------|
| `REDIS_HOST` | `redis.example.edu.cn` | Redis 主机 |
| `REDIS_PORT` | `6379` | Redis 端口 |
| `REDIS_PASSWORD` | （随机强密码） | Redis 密码，无密码可留空 |
| `REDIS_DATABASE` | `0` | Redis 库索引 |

### 2.4 安全与微信

| 变量名 | 示例 | staging | prod | 说明 |
|--------|------|---------|------|------|
| `JWT_SECRET` | 至少 32 字符随机串 | **必填** | **必填** | JWT 签名密钥，**禁止** dev 默认值 |
| `JWT_EXPIRE_DAYS` | `7` | 可选 | 可选 | Token 有效期（天） |
| `WX_APPID` | 微信小程序 AppID | 真机验收**建议填** | **必填** | 启动时 prod 强制校验；staging 允许为空 |
| `WX_SECRET` | 微信小程序 Secret | 真机验收**建议填** | **必填** | `dev-mode=false` 且为空时，wx-login 运行时报错 |
| `WX_TMPL_ENROLL_SUCCESS` | 报名成功订阅消息模板 ID | 可选 | 建议填 | dev 可空，走站内消息兜底 |
| `WX_TMPL_ENROLL_APPROVED` | 报名审核通过模板 ID | 可选 | 可选 | |
| `WX_TMPL_ACTIVITY_REMIND` | 活动提醒模板 ID | 可选 | 可选 | |

**staging 定位说明：**

| 验收场景 | WX 凭证是否必须 |
|----------|----------------|
| 后端 API + 学号登录 + 管理端 | 否（可启动、可测大部分链路） |
| 体验版 / 真机微信登录验收 | **是**（须配置 `WX_APPID` / `WX_SECRET`） |

### 2.4.1 CORS（前后端分域部署时）

| 配置项 | 说明 |
|--------|------|
| `SHUYUAN_CORS_ALLOWED_ORIGIN_PATTERNS` | 逗号分隔的管理后台域名，映射为 `shuyuan.cors.allowed-origin-patterns` |
| `shuyuan.cors.allowed-origin-patterns` | 亦可在 `application-prod.yml`（部署机本地）中写入 |

**默认推荐同源反代**（Nginx 同域提供 `/api/` 与 `/admin/`），多数部署**无需**跨域，白名单可留空。

staging/prod **禁止**在白名单中包含 `localhost`、`127.0.0.1`、`example.edu.cn` 占位域或通配符 `*`。未配置白名单时，跨域请求将被拒绝（同源部署不受影响）。

### 2.5 阿里云 OSS / CDN（媒体主链，staging / prod 建议配置）

| 变量名 | 示例 | 说明 |
|--------|------|------|
| `OSS_ENABLED` | `true` | 是否启用 OSS 上传与签名 URL |
| `OSS_ENDPOINT` | `https://oss-cn-guizhou.aliyuncs.com` | OSS 地域 Endpoint |
| `OSS_BUCKET` | `shuyuan-assets` | Bucket 名称 |
| `OSS_ACCESS_KEY` | （子账号 AK） | 仅限对应 Bucket 权限 |
| `OSS_SECRET_KEY` | （子账号 SK） | 与 AK 配对 |
| `OSS_CDN_DOMAIN` | `https://cdn.shuyuan.example.com` | CDN 加速域名（可选） |
| `OSS_SIGN_EXPIRE_SECONDS` | `7200` | 签名 URL 有效期（秒） |
| `OSS_MAX_UPLOAD_BYTES` | `209715200` | 后台中转上传单文件上限（字节） |

未配置 OSS 时：开发环境可手填公开 CDN 地址；生产需配置私有 Bucket + 签名下发（验收 §六）。

### 2.6 登录防刷（可选）

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| `LOGIN_MAX_FAIL_ATTEMPTS` | `5` | 连续失败锁定阈值 |
| `LOGIN_LOCK_MINUTES` | `5` | 锁定时长（分钟） |
| `LOGIN_FAIL_WINDOW_MINUTES` | `15` | 失败计数窗口（分钟） |

### 2.6 测试专用（test profile / CI）

|--------|--------|------|
| `TEST_DB_URL` | 本地 shuyuan_test | 测试库连接串 |
| `TEST_DB_USERNAME` | `root` | 测试库用户名 |
| `TEST_DB_PASSWORD` | `dev123456` | 测试库密码 |
| `TEST_REDIS_HOST` | `localhost` | 测试 Redis 主机 |
| `TEST_REDIS_PORT` | `6379` | 测试 Redis 端口 |
| `TEST_REDIS_PASSWORD` | `dev123456` | 测试 Redis 密码 |
| `TEST_REDIS_DATABASE` | `1` | 测试 Redis 库（与 dev 隔离） |

### 2.7 OpenAPI 文档（E2-4）

| Profile | Swagger UI | 说明 |
|---------|------------|------|
| dev / docker / staging | 开启 | `/swagger-ui.html`、`/v3/api-docs` |
| prod | **关闭** | 见 `application-prod.example.yml` |

详见 `docs/api/README.md`。

### 2.8 接口限流（E2-3）

| 配置项 | 默认 | 说明 |
|--------|------|------|
| `shuyuan.rate-limit.enabled` | `true` | 是否启用 |
| `login-per-minute` | 10 | 学号登录，每 IP |
| `wx-login-per-minute` | 20 | 微信登录，每 IP |
| `admin-login-per-minute` | 10 | 管理端登录，每 IP |
| `enroll-per-minute` | 5 | 活动报名，每用户 |
| `ai-per-day` | 20 | AI 问答（预留），每用户每天 |

超限返回 HTTP **429** + `body.code: 429`（与 `ApiHttpStatus` 对齐）。

### 2.9 运维告警（E2-1）

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| `ALERT_ENABLED` | `false` | 是否启用 5 分钟告警任务 |
| `ALERT_WEBHOOK_URL` | 空 | 钉钉 / 企业微信机器人 Webhook |
| `ALERT_WINDOW_MINUTES` | `5` | 错误率统计窗口（与任务周期一致） |
| `ALERT_ERROR_RATE_PERCENT` | `1` | 5xx 占比阈值（%） |
| `ALERT_MIN_SAMPLE_SIZE` | `20` | 窗口内最少请求数 |
| `ALERT_COOLDOWN_MINUTES` | `15` | 同类告警冷却（分钟） |

详见 `docs/运维/运维手册_V1.0.md` §3。

## 3. 配置文件与密钥管理

### 3.1 仓库内文件

```
backend/src/main/resources/
  application.yaml              # 公共配置，profile 默认 dev
  application-dev.yml           # 本地开发（含 dev 密码，仅本地/演示）
  application-docker.yml        # Docker Compose
  application-staging.yml       # 预发（无明文密码）
  application-prod.example.yml  # 生产模板（无真实密钥）

backend/src/test/resources/
  application-test.yml          # 测试 profile 骨架

.env.example                    # 根目录变量示例
```

### 3.2 禁止提交

以下文件已在 `.gitignore` 中，**不得** push 到远端：

- `.env` / `.env.*`
- `application-local.yml`
- `application-prod.yml`（含真实 prod 密钥）

### 3.3 生产部署建议

1. 复制 `application-prod.example.yml` 为部署机本地的 `application-prod.yml`，或通过 K8s Secret / 环境变量注入。
2. 设置 `SPRING_PROFILES_ACTIVE=prod`。
3. 确认 `shuyuan.wx.dev-mode=false`（模板已默认关闭）。
4. 确认日志级别为 `INFO`/`WARN`，无 DEBUG、无 SQL stdout。

## 4. 各环境差异摘要

### dev

- 本地 MySQL/Redis，`dev123456` 密码可接受。
- MyBatis SQL 打印到控制台。
- `com.shuyuan` 日志级别 `DEBUG`。
- 微信 `dev-mode=true`，code 直接换 token。

### docker

- 数据库/Redis 使用 compose 服务名 `mysql` / `redis`。
- SQL 日志关闭，日志 `INFO`。

### staging

- 全部敏感项从环境变量读取。
- 禁止 dev 密码、禁止 mock / dev-mode。
- 日志 `INFO`，SQL 关闭。
- API 域名示例（占位，部署前必须替换）：`https://staging.example.edu.cn/api/v1`
- **微信凭证**：启动不强制，但真机 `wx-login` 验收前须配置 `WX_APPID` / `WX_SECRET`。
- **CORS**：`application-staging.yml` 已含示例白名单；同源反代部署可不改。

### prod

- 同 staging，日志更严格（root `WARN`）。
- API 域名示例（占位）：`https://example.edu.cn/api/v1`
- 上线门禁：无 dev 密码、无 DEBUG、无 mock fallback（小程序 PR3 门禁）。

## 5. CI 与环境的关系

GitHub Actions（`.github/workflows/ci.yml`）当前执行：

| Job | 命令 |
|-----|------|
| Backend | `mvn -B test`（JDK 17；使用 Actions 自带 Maven，非 wrapper） |
| Admin | `npm ci && npm run typecheck && npm run build` |
| Miniapp Mock Guard | `npm run check:prod-mock` |
| 部署环境门禁（发布前本地） | `npm run check:deploy-env -- --env /path/to/.env --release` |

> 根目录另有 `npm run check:release-env`（小程序 `--release` 门禁）与 `npm run check:deploy-env`（后端 `.env` 门禁），**发布前**须本地执行；尚未接入 CI。  
> **注意：** 在 `ENV=dev` 下执行 `check:release-env` **失败是预期行为**（要求 `ENV=prod` 与正式域名）；日常开发与 CI 请使用 `check:prod-mock`。

## 8. E1 可观测（PR2）

### 健康检查 `GET /api/v1/health`

响应示例（`data` 字段内）：

```json
{
  "status": "UP",
  "db": "UP",
  "redis": "UP",
  "version": "0.0.1-SNAPSHOT",
  "profile": "dev",
  "time": "2026-07-06T00:00:00"
}
```

- `APP_VERSION` 环境变量可覆盖 `version`
- `APP_COMMIT` 可选，CI 可注入 git short SHA
- DB/Redis 异常时对应字段为 `DOWN`，整体 `status` 为 `DOWN`

### TraceId

- 请求头 `X-Trace-Id`：客户端可传入，否则服务端自动生成
- 响应头回传同一 `X-Trace-Id`
- 日志 pattern 含 `%X{traceId}`，可按 traceId 串联单次请求
