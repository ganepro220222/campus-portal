# 云端书院 · 管理员操作手册 V1.0

> 贵州交通职业大学 ·「马院+书院」协同育人项目  
> 对照：《交付物与验收标准 V1.2》§一 交付物 §7  
> 适用对象：书院运营人员、内容管理员、系统管理员  
> 最后更新：2026-07-08

本文档说明管理后台各模块的日常操作流程、权限说明与常见问题。后台地址示例：`https://your-domain.com/admin/`（本地开发为 `http://localhost:5173`）。

---

## 1. 登录与权限

### 1.1 登录

1. 打开管理后台登录页  
2. 输入管理员账号与密码  
3. 登录成功后进入「首页概览」  

**开发环境默认账号：** `admin` / `Admin@123`（生产环境须由系统管理员重置密码，禁止使用默认密码上线。）

### 1.2 登录安全

- 连续 **5 次**密码错误，账号锁定 **5 分钟**  
- 锁定期间提示「请稍后再试」，属正常防护  
- 忘记密码请联系系统管理员在数据库或运维流程中重置  

### 1.3 权限说明

| 权限标识 | 能力范围 |
|----------|----------|
| `admin:super` | 超级管理员，拥有全部菜单与操作 |
| `news:read` | 查看新闻 |
| `news:write` | 新建/编辑新闻 |
| `news:publish` | 发布/下架新闻 |
| `hall:read` | 查看展馆、文创 |
| `hall:write` | 编辑展馆、文创 |
| `course:read` | 查看课程、资源 |
| `course:write` | 编辑课程、资源 |
| `enroll:read` | 查看活动、报名列表 |
| `enroll:export` | 导出报名 Excel |
| `stats:view` | 查看首页数据看板 |

侧栏仅显示当前账号有权限的菜单。无权限访问某 URL 会自动跳回首页。

---

## 2. 首页概览

登录后默认页面，包含：

- **欢迎信息**：当前登录管理员与角色  
- **数据看板**（需 `stats:view` 或超管）：  
  - KPI：今日访问、累计会员、内容总量等  
  - 近 30 日访问趋势图  
  - 各板块访问分布  
  - 内容浏览排行  
  - **导出月报**：选择月份后下载统计 Excel  

若看不到数据看板，说明账号无 `stats:view` 权限，联系管理员分配。

## 3. 新闻管理

**路径：** 侧栏 → 新闻管理  
**权限：** 查看 `news:read`；编辑 `news:write`；发布 `news:publish`

### 3.1 新建新闻

1. 点击「新建新闻」  
2. 填写标题、分类、封面、摘要、正文  
3. 可选：勾选「置顶」  
4. 保存后为**草稿**状态，小程序端不可见  

### 3.2 发布与下架

| 操作 | 说明 |
|------|------|
| 发布 | 草稿 → 已发布，小程序端立即可见，并同步搜索索引 |
| 下架 | 已发布 → 草稿，小程序端不再展示 |

发布、下架前系统会弹出确认框，请核对标题后再确认。

### 3.3 筛选与维护

- 按状态（草稿/已发布）、分类筛选  
- 「阅读」列为累计浏览量（Redis 定期同步）  
- 已发布内容若需修改，编辑后保存即可，无需重新发布  

---

## 4. 展馆管理

**路径：** 侧栏 → 展馆管理  
**权限：** `hall:read` / `hall:write`

维护 11 个文化展馆的基础信息：名称、简介、封面、排序、上下架状态。

| 状态 | 小程序表现 |
|------|------------|
| 上架 | 展馆列表与详情可见 |
| 下架 | 不可见 |

展馆内展品、语音讲解等内容在编辑表单中维护。大图片、音频建议通过 OSS 上传组件上传；未配置 OSS 时可粘贴已有 CDN 地址。

---

## 5. 文创管理

**路径：** 侧栏 → 文创管理  
**权限：** 同展馆（`hall:read` / `hall:write`）

维护文创产品双语介绍、鉴赏图、3D 展示链接、咨询方式等。上下架规则与展馆一致。

---

## 6. 课程管理

**路径：** 侧栏 → 课程管理  
**权限：** `course:read` / `course:write`

### 6.1 基本流程

1. 新建课程，填写标题、讲师、简介、封面  
2. 配置视频地址（OSS 上传或粘贴 URL）  
3. 设置上下架：仅**上架**课程在小程序「在线课程」中展示  
4. 可选：配置 AI 字幕相关字段（若已生成字幕）  

### 6.2 媒体上传

- 已配置 OSS：表单内使用上传组件，自动回填地址  
- 未配置 OSS：向运维索取 CDN 地址后手动填入  

视频文件较大，**优先直传 OSS**，避免长时间占用应用服务器带宽。

---

## 7. 资源管理

**路径：** 侧栏 → 资源管理  
**权限：** `course:read` / `course:write`

管理 PDF、音视频等可下载资料。字段包括标题、类型、文件地址、上下架状态。下架后小程序不可下载。

---

## 8. 活动管理

**路径：** 侧栏 → 活动管理  
**权限：** `enroll:read`；编辑需写权限；导出需 `enroll:export`

### 8.1 新建与发布活动

1. 点击「新建活动」  
2. 填写名称、地点、时间、名额（0 表示不限）、封面、详情  
3. **报名审核**开关：  
   - **关闭**：报名即成功占位  
   - **开启**：用户提交后为「待审核」，需管理员通过后才占位  
4. 保存为草稿，确认无误后点击「发布」  

发布后小程序「活动报名」列表可见。

### 8.2 名额与并发

- 列表「名额」列显示 `已报名/总名额`  
- 名额满后小程序提交报名会被拒绝（系统有数据库唯一约束与并发保护）  
- 取消活动后不再接受新报名  

### 8.3 报名管理

在活动列表点击「报名管理」进入该活动的报名明细页。

| 功能 | 说明 |
|------|------|
| 筛选 | 按状态：待审核 / 已通过 / 已拒绝 / 已取消 |
| 审核通过 | 仅「需审核」活动且状态为待审核时可用 |
| 拒绝 | 填写原因后拒绝该报名 |
| 导出 Excel | 导出当前活动全部报名记录（姓名、手机、学院等） |

导出文件可用于线下签到或存档。

## 12. AI 知识库

**路径：** 侧栏 → AI 知识库  
**权限：** `admin:super`

为小程序「AI 文化问答」提供资料来源。

### 12.1 录入资料

1. 点击「录入资料」  
2. 填写标题与正文（建议 500 字以上，内容准确、来源可靠）  
3. 点击「入库」  

系统自动将正文切分为多个片段写入知识库。

### 12.2 状态说明

| 状态 | 含义 |
|------|------|
| 已就绪 | 切分完成，可被 AI 检索引用 |
| 处理中 | 刚入库，稍候刷新 |

### 12.3 使用建议

- 按主题分篇录入（如「阳明心学概要」「屯堡文化简介」），便于检索命中  
- 删除文档后，相关片段一并移除，AI 将不再引用  
- 小程序用户每日问答次数有限（默认 20 次/人），属正常限流  

### 12.4 AI 回答机制

- 配置了智谱 API Key：基于知识库片段 + 大模型生成回答  
- 未配置 Key：基于检索片段自动组织回答（效果略简）  
- 违规内容会被安全审核拦截，用户看到「请修改后重试」类提示

## 13. 学院矩阵

**路径：** 侧栏 → 学院矩阵  
**权限：** `admin:super`

配置贵州交职大各二级学院在小程序「学院矩阵」页的展示与对接方式。

### 13.1 对接方式

| 方式 | 适用场景 | 必填项 |
|------|----------|--------|
| 手动录入 | 对方提供图文简介 | 学院名称、简介 |
| 小程序跳转 | 对方有独立小程序且已关联 | AppID、页面路径（可选） |
| H5 嵌入 | 对方有已备案 HTTPS 网页 | 内容地址（https） |
| 接口同步 | 对方提供开放 API（预留） | 内容地址、Token（可选） |

### 13.2 操作步骤

1. 点击「新增学院」或编辑已有条目  
2. 填写学院名称、简介、排序  
3. 选择对接方式并补全对应字段  
4. 设为「上架」后保存  

小程序端按排序展示；下架后立即隐藏。

### 13.3 注意事项

- **小程序跳转**：须在 `miniapp/app.json` 的 `navigateToMiniProgramAppIdList` 中预先声明目标 AppID（最多 10 个），并在微信公众平台完成双方关联  
- **H5 嵌入**：须在小程序后台配置业务域名白名单  
- 开发环境首次启动若列表为空，可重置 Docker 数据库以导入 `seed-dev.sql` 中的 11 条演示数据

## 14. 操作日志

**路径：** 侧栏 → 操作日志

## 15. 内容发布通用规范

### 15.1 发布前检查

- [ ] 标题、封面、正文无错别字与敏感表述  
- [ ] 图片清晰，宽高比适合手机端  
- [ ] 视频/音频可正常播放（先自己在小程序预览）  
- [ ] 活动时间、地点、名额正确  
- [ ] 外链 https 可访问  

### 15.2 下架场景

- 新闻：信息过时、内容需更正  
- 展馆/课程/资源：维护中、版权到期  
- 活动：因故取消（使用「取消活动」而非仅下架）  
- Banner/公告：活动结束  

下架后小程序端通常**立即**不可见，无需额外操作。

### 15.3 预览建议

重大内容发布前：

1. 后台保存为草稿  
2. 使用测试学员账号在小程序登录预览  
3. 确认无误后再点「发布」

## 16. 常见问题

| 问题 | 原因与处理 |
|------|------------|
| 登录提示锁定 | 密码多次错误，等待 5 分钟或联系管理员清 Redis 锁 |
| 看不到某菜单 | 账号无对应权限，联系超管分配 |
| 发布后小程序看不到 | 检查是否真为「已发布/上架」；小程序下拉刷新；确认非 mock 环境 |
| 图片上传失败 | OSS 未配置或文件过大，联系运维或改用 CDN 链接 |
| 报名导出为空 | 确认活动已有报名记录；检查 `enroll:export` 权限 |
| AI 回答不准确 | 补充或修正知识库资料；问题尽量具体 |
| 学院矩阵为空 | 后台未配置或未上架；开发环境可重置 DB 导入种子数据 |
| 小程序跳转失败 | 检查 AppID 是否写入 app.json 且双方小程序已关联 |
| 数据看板无数据 | 需有用户访问后才会积累；新环境初期为 0 属正常 |

## 17. 角色分工建议

| 角色 | 建议权限 | 职责 |
|------|----------|------|
| 超级管理员 | `admin:super` | 账号管理、Banner/公告、知识库、学院矩阵、日志审计 |
| 内容编辑 | `news:write` + `hall:write` + `course:write` | 日常内容录入，发布权可单独授予 |
| 内容审核 | `news:publish` | 新闻发布/下架终审 |
| 活动运营 | `enroll:read` + `enroll:export` | 活动发布、报名审核、导出 |
| 数据分析 | `stats:view` | 查看看板、导出月报 |

具体账号创建与权限分配由系统管理员在数据库 `admin_user` 表或后续账号管理功能中维护（当前版本通过种子数据或 SQL 调整 `permissions` 字段）。
