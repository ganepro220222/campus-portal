> 对照：《交付物与验收标准 V1.2》§一 交付物 §9、§七

本文档列出项目上线与运维所需的全部账号类型、交接内容、权限边界与安全规则。**实际账号、密码、API Key 不得写入 Git 仓库**，应通过线下安全渠道移交校方。

## 2. 各账号详细说明

### 2.1 微信小程序后台

| 项 | 说明 |
|----|------|
| 登录入口 | [微信公众平台](https://mp.weixin.qq.com/) |
| 必要权限 | 管理员（开发、数据分析、用户反馈）；校方经办人建议同为管理员 |
| 需交接信息 | AppID、AppSecret（**禁止明文微信/邮件传递**）、已配置合法域名列表 |
| 域名配置 | request / upload / download / socket 合法域名须与 API、OSS、CDN 一致 |
| 订阅消息 | 报名成功等模板 ID，填入环境变量 `WX_TMPL_ENROLL_SUCCESS` 等（见 `docs/运维/环境变量说明.md`） |
| 关联小程序 | 兄弟小程序跳转需在「设置 → 第三方设置」配置 `navigateToMiniProgramAppIdList` |

**开发环境：** `application-dev.yml` 中 `wx.dev-mode=true`，无需真实 AppID 即可本地调试登录。

**生产环境：** 必须 `wx.dev-mode=false`，配置真实 `WX_APPID` / `WX_SECRET`。

### 2.2 云服务器（ECS）

| 项 | 说明 |
|----|------|
| 用途 | Nginx（HTTPS）、Spring Boot API、管理后台静态资源 |
| 建议规格 | 4C8G（见 `docs/运维/资源配置清单_1.3万学生.md`） |
| 交接方式 | SSH 私钥文件（加密压缩包）或限时密码 + 首次登录强制改密 |
| 安全要求 | 安全组仅开放 443（及 80→443 跳转）；**禁止**公网开放 3306 / 6379 |
| 部署参考 | `docs/运维/部署手册_V1.0.md` §4–§5 |

### 2.3 数据库（MySQL 8.0）

| 项 | 说明 |
|----|------|
| 建议形态 | 云 RDS，自动日备开启，保留 30 天 |
| 应用账号 | `shuyuan_app`：仅 DML + 必要 DDL（通过 patch 脚本评审） |
| 运维账号 | 高权限账号仅限运维本机或堡垒机访问 |
| 连接配置 | 环境变量 `DB_URL` / `DB_USERNAME` / `DB_PASSWORD` |
| 初始化 | `sql/init.sql` + 按需 `sql/patch-*.sql`（见 `sql/README.md`） |

### 2.4 Redis 7

| 项 | 说明 |
|----|------|
| 用途 | 登录锁、接口限流、浏览量计数、DAU、积分每日上限 |
| 连接配置 | `REDIS_HOST` / `REDIS_PORT` / `REDIS_PASSWORD` / `REDIS_DATABASE` |
| 数据特性 | 缓存类数据可丢失；浏览量有 5 分钟落库兜底（`ViewCountFlushJob`） |

### 2.5 阿里云 OSS 与 CDN

| 项 | 说明 |
|----|------|
| 用途 | 课程视频、展馆图片、文创模型、课件等资源存储与分发 |
| 权限模型 | **子账号** AK/SK，仅限对应 Bucket 读写；生产 Bucket **私有读** + 签名 URL |
| 环境变量 | `OSS_ENABLED`、`OSS_ENDPOINT`、`OSS_BUCKET`、`OSS_ACCESS_KEY`、`OSS_SECRET_KEY`、`OSS_CDN_DOMAIN` |
| 验收要求 | 直接访问 OSS 原始地址应 403；小程序通过签名 URL 可播放（验收 §六） |

### 2.6 域名与 HTTPS

| 项 | 说明 |
|----|------|
| API 域名 | 例：`api.shuyuan.example.edu.cn` → Nginx → Spring Boot :8080 |
| 管理后台 | 例：`admin.shuyuan.example.edu.cn` 或同域 `/admin/` |
| 证书 | Let's Encrypt 或校方 CA；须在微信公众平台配置为合法域名 |
| 强制 HTTPS | 验收 §六 硬性要求 |

### 2.7 AI 平台

| 项 | 说明 |
|----|------|
| 用途 | AI 文化问答、后台文案润色 |
| 建议 | 使用**学校主体**申请的 API Key，便于合规与续费 |
| 环境变量 | `AI_API_KEY`、`AI_BASE_URL`、`AI_MODEL`（见环境变量说明 §2.9） |
| 安全 | Key 仅存在于服务器环境变量，**不得**提交至 Git 或写入前端 |

### 2.8 管理后台超管

| 环境 | 账号 | 说明 |
|------|------|------|
| 开发 / Docker | `admin` / `Admin@123` | 仅本地调试，**禁止用于生产** |
| 生产 | 独立超管（校方指定） | 上线前新建账号，执行 `sql/patch-admin-account-security.sql` 后**禁用默认 admin** |

首次登录后须立即修改密码；连续 5 次错误锁定 5 分钟（Redis）。

## 3. 密钥交接安全规则

对照《交付物与验收标准 V1.2》§七：

1. **禁止**通过微信群、普通邮件、文档附件明文传递 API Key、数据库密码、JWT Secret。  
2. **推荐**方式：加密压缩包（密码电话/当面告知）、校方密码管理器、线下当面交接。  
3. 交接完成后，乙方侧开发用密钥应**轮换或作废**。  
4. 生产环境 `JWT_SECRET` 至少 32 字符随机串，与 dev 默认值完全不同。  
5. 仓库内仅保留 `application-prod.example.yml` 占位，真实 `application-prod.yml` 不入库。

## 6. 源代码仓库交接

| 项 | 说明 |
|----|------|
| 仓库 | 私有 Git 仓库（GitHub / 校内 GitLab，以实际为准） |
| 主分支 | `main` — 可发布分支，CI 全绿方可合并 |
| 工作分支 | `work` / feature 分支 — 开发完成后 PR 合并至 `main` |
| CI | `.github/workflows/ci.yml`：backend `mvn test`、admin typecheck+build、miniapp mock 门禁 |
| 版本 Tag | 建议 SemVer：`v1.0.0-rc1`、`v1.0.0`；Tag 与线上部署版本一一对应 |
| 目录 | `backend/`、`admin/`、`miniapp/`、`sql/`、`docs/`、`scripts/` |

**交接动作：**

1. 校方 Git 账号加入仓库 **Maintainer** 或转移仓库所有权。  
2. 提供只读备份镜像（可选）。  
3. 附 `CHANGELOG.md`（发布时）与各 Tag 部署记录。
