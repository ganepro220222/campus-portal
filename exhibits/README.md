# 云端书院 · 增强版 3D 鉴赏系统

对方案 `docs/三维展示系统/增强版3D鉴赏系统_实施方案_v2.md` 的**可运行实现**。
纯前端，自托管 Three.js（`vendor/`，**不依赖任何外部 CDN**）。

## 架构：共享播放器 + 数据分离（应对 100+ 展品）
```
exhibits/
  player.html          # 播放器 + 编辑器（全站唯一一份）
  vendor/              # Three.js + Draco/Basis 解码器（全站唯一一份）
  studio.html          # 开发方工作台：列出所有展品，点卡片直达
  manifest.json        # 展品目录清单（有后端后可改为自动扫目录）
  craft-001/
    config.json        # 该展品的全部配置（标题/相机/材质/热点/光照/语音…）
    assets/            # model.glb + panorama.jpg + poster.jpg + 音频
    index.html         # ~15 行跳转壳 → ../player.html?ex=craft-001
  craft-002/ …         # 每件展品只是「一份 config + assets」
```
- **一份代码带上百个展品**：升级 Three.js / 修编辑器 bug 只改一处，全站生效；vendor 只存一份（省掉每包 ~2.9MB 重复）。
- 新增展品 = 复制一个 `craft-XXX/` 目录改数据 + 在 `manifest.json` 加一行 → 工作台自动出现。

## 本地运行
```bash
cd exhibits
python3 -m http.server 8099
# 工作台（推荐）： http://127.0.0.1:8099/studio.html
```
> 必须用「静态服务器」打开（ES 模块 + fetch 需 http 协议，不能双击）。

## 入口一览
| 用途 | 地址 | 谁用 |
|---|---|---|
| 工作台（所有展品入口） | `…:8099/studio.html` | 开发/维护方 |
| 观看某展品 | `…/player.html?ex=craft-001` 或 `…/craft-001/` | 终端用户 / VR 平台外链 |
| 编辑某展品 | `…/player.html?ex=craft-001&mode=edit` | 开发/维护方 |

### 编辑版面板（接近方案 v2 第 5 节 + 需求增补 v3）
- **基本信息**：工艺品名称 / 副标题（实时更新左上角）。
- **用户端按钮**：勾选终端用户可见的功能按钮（自动旋转 / 热点 / 重置 / 全屏 / 光源预设）。
- **资产**：从**任意 URL 加载模型**（相对/绝对；跨域需 CORS）、拖拽/选择本地 `.glb` 预览、显示当前模型路径。
- **模型 Transform**：缩放 / 适配尺寸 / 位移 XYZ / 旋转 Y°，可复位模型。
- **相机**：FOV / 自动旋转速度 / 最近最远距离 / **旋转轴 Y 偏移**（默认 0 = 模型中轴）/ **保存当前视角为默认**。
- **材质**：全局 曝光 / 环境光强 / 金属度 / 粗糙度；**按材质名分组覆盖**（自动列出模型材质名）。
- **灯光（四光源）**：环境 / 主 / 补 / 轮廓光 的强度与颜色。
- **环境 IBL**：可见全景背景开关、换全景图 URL。
- **热点与面板样式**：信息面板样式（黑底 / 毛玻璃 / 透明）、热点颜色 / 大小 / 脉冲开关。
- **热点**：Shift+点击模型表面新增；**直接拖拽热点微调点位**、位置 X/Y/Z 数字微调；逐条改标题/文案、删除、**更新聚焦机位（存完整角度+距离）**、**绑定语音**。
- **语音讲解**：新增 / 删除音频、改名称与地址；多条自动出现下拉切换。
- **预设（含灯光方案）**：从当前状态新建（记录曝光/环境光强/背景 **+ 四光源灯组**）/ 删除；每个预设可选**是否显示为前台按钮**，前台**最多 4 个**。
- **性能**：FPS / draw calls / 三角面 / 顶点 / 材质 / 贴图 / DPR，超阈值告警。
- **导出**：导出 `config.json`、**导出仅观看版 `index.view.html`（剔除编辑器代码，前台更轻更安全）**、截取当前帧为 `poster.jpg`、校验 + 部署清单（可折叠）。

> 所有可调滑条均配**数字输入框**，可直接键入精确值。

### 交互细节（需求增补 v3）
- **3D 引线标注**：交互热点（双环标记，任意底色可见，可设色）→ 点击后**桌面**引线牵引信息面板落在模型投影外的留白区（不遮挡器物）；**移动端**面板就近显示、夹取在视口内、不遮挡工具条（无引线）。
- **信息面板样式**：黑底 / 毛玻璃 / 透明底可切换。
- **语音讲解**：左上角小型播放器（播放/暂停 + 进度 + 时长），多条下拉切换；热点可绑定音频点击自动播放。
  - 样片内 `assets/narration-1.wav`、`narration-2.wav` 为**占位示例音**（柔和提示音，非真人讲解），正式上线由校方替换为真人讲解 `.mp3`。

> 关于「模型放哪」：默认样片是**同目录静态包**（`assets/model.glb`，最省心、免 CORS）。但 `config.json` 的 `assets.model` 可填**任意 URL**——编辑版「资产 → 从 URL 加载」即演示了这点；跨域资源需服务器配 CORS。

## 已演示的方案要点
- **真实全景 IBL**：`assets/panorama.jpg`（由 `test/4/BJ.jpg` 50MB 压到 **497KB / 2048×1024**）经 PMREM 作环境反射，釉面高光"活"起来（无全景时自动降级 RoomEnvironment）。
- **配置驱动**：所有模型/相机/材质/热点/文案/预设都在 `config.json`，一份 `index.html` 通用。
- **热点讲解**：点击热点 → 暂停旋转 + 相机平滑聚焦 + 讲解卡片（PC 右侧栏 / 手机底部抽屉）+ 双语；转到背面自动淡化。
- **加载编排**：poster 占位 → 模型淡入 → 全景 IBL 后台接管；WebGL 不支持 / 加载失败有降级与重试。
- **交付闭环**：开发方在编辑版调参打点 → 导出 `config.json` → 替换展包 → 上传静态服务器。校方不接触 GLB / 材质 / 热点。

### 无障碍 / 性能 / 兼容
- **键盘快捷键**：`Tab` 在热点/按钮间移动（焦点高亮）、`Enter/空格` 打开聚焦的热点、`Esc` 关闭讲解面板、`空格` 播放/暂停语音讲解。
- **无障碍 (a11y)**：热点/按钮具备 `role` / `aria-label` / `aria-pressed`；背面隐藏的热点自动移出 Tab 序；面板 `role="dialog"`；`prefers-reduced-motion` 下停用脉冲/自动旋转。
- **压缩模型支持**：GLTFLoader 已挂载 **Draco（几何压缩）+ KTX2/Basis（贴图压缩）** 解码器（自托管于 `vendor/draco`、`vendor/basis`，无 CDN）；仅当模型实际使用压缩时才按需拉取解码器，普通模型零开销。
- **移动端像素比自适应**：手机端限流到 `performance.mobilePixelRatio`（默认 1.5）省电控温，桌面端放开到 `desktopPixelRatio`（默认 2）保清晰。

## 新增一件展品
```
exhibits/craft-XXX/
  config.json     # 复制一份改：标题/模型/相机/材质/热点/光照/语音…
  assets/         # 放 model.glb + panorama.jpg（2:1 等距柱状）+ poster.jpg + 音频
  index.html      # ~15 行跳转壳（把 craft-001/index.html 改个 ex 即可）
```
再在 `manifest.json` 的 `exhibits` 数组加一行目录名 → 工作台自动出现。**不复制 vendor、不复制播放器代码。**

## 上线注意（详见方案 v2 第 10/12 节）
- **公开路径**：部署共享 `player.html` 的**仅观看版**（编辑器里「导出仅观看版」生成 `player.view.html`）+ 各展品数据目录；VR 平台热点外链指向 `…/craft-001/` 或 `…/player.html?ex=craft-001`。
- **编辑路径**：完整 `player.html` + `studio.html` 放**受保护地址**（Basic Auth / IP 白名单），仅开发/维护方可达。
- 全站静态（player/vendor/展品数据）建议托管 **OSS + CDN**；`config.json` **不走 CDN 缓存或带版本号**，避免编辑保存后看不到更新。
- 资源建议带版本号或 hash 以避免浏览器缓存。
